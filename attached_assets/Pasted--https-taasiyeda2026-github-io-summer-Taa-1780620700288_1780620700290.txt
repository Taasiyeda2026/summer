יש לבצע תיקון ממוקד בתבנית הקורסים בלבד באתר:
https://taasiyeda2026.github.io/summer/

ריפו:
Taasiyeda2026/summer

קבצים רלוונטיים:
course-page.html
catalog-data.json
service-worker.js אם קיים

מטרה:
לעדכן את תצוגת “תוצרי סיום” ו“כלים ומיומנויות” כך שתיראה כמו הדוגמה שסופקה:
שורת תיבות קומפקטיות לתוצרי הסיום, ומתחתיה אזור כרטיסיות מסודר של קבוצות מיומנויות.

אין לשכתב את כל העמוד.
אין לשנות את מבנה האתר הכללי.
אין לפגוע בתצוגת תוכניות קיימות.
אין להסיר שדות קיימים.
יש לבצע שינוי נקי, ממוקד ותואם לעיצוב הקיים.

1. תוצרי סיום

השדה הקיים outcome צריך להמשיך לעבוד.

אם outcome כולל כמה שורות, למשל:
"אב-טיפוס ביומימטי\nהדמיה דיגיטלית\nפוסטר חקר מדעי\nפיץ׳ יזמי קצר\nפרזנטציה מסכמת"

יש להציג כל שורה כתיבה נפרדת, קומפקטית ומעוצבת, באותה שורה או בשבירה טבעית לפי רוחב העמוד.

עיצוב נדרש:
- תיבות בהירות ועדינות
- גבול דק
- טקסט מודגש וברור
- גובה נמוך יחסית
- ריווח אחיד
- התאמה מלאה ל־A4
- ללא עומס חזותי

2. כלים ומיומנויות

להוסיף תמיכה בשדה חדש ב־catalog-data.json בשם:

skillGroups

מבנה השדה:

"skillGroups": [
  {
    "title": "חקר ומידע",
    "items": ["אוריינות מדעית", "אוריינות מידע", "שאילת שאלות"]
  },
  {
    "title": "חשיבה וחדשנות",
    "items": ["חשיבה ביקורתית", "חשיבה יצירתית", "פתרון בעיות"]
  },
  {
    "title": "דיגיטל וטכנולוגיה",
    "items": ["אוריינות דיגיטלית", "הדמיה", "שימוש בכלי AI"]
  },
  {
    "title": "עבודה והצגה",
    "items": ["עבודת צוות", "קבלת משוב", "הצגה בפני קהל"]
  }
]

בתצוגה:
- כל קבוצה תוצג ככרטיס נפרד.
- בראש כל כרטיס תופיע כותרת הקבוצה.
- מתחתיה יופיעו הפריטים בשורות קצרות.
- הכרטיסים יהיו בשורה אחת של 4 כרטיסים כאשר יש מקום.
- אם אין מקום, תהיה שבירה מסודרת.
- הכרטיסים צריכים להיות קומפקטיים, נקיים, עם גבול עדין ורקע בהיר.
- הטקסט צריך להיות קריא ולא צפוף מדי.
- לא להשתמש באייקונים גדולים או בעיצוב עמוס.

3. שמירת תאימות לאחור

אם אין skillGroups בתוכנית מסוימת:
- להשאיר את ההתנהגות הקיימת.
- כלומר להשתמש ב־skills אם הוא מערך.
- אם skills אינו מערך, להשתמש ב־develops כפי שהקוד עושה כיום.
- לא לשבור תוכניות קיימות.

4. עדכון normalizeProgram

ב־course-page.html יש לעדכן את normalizeProgram כך שיקרא גם:

skillGroups

ולוודא שהוא מערך תקין בלבד.
אם השדה לא קיים או לא תקין, להחזיר מערך ריק.

5. עדכון buildSkillsGroups

לעדכן את הפונקציה buildSkillsGroups(p):

אם p.skillGroups קיים ויש בו קבוצות:
- להחזיר HTML של כרטיסיות קבוצות מיומנויות.

אם לא:
- להחזיר את התצוגה הקיימת של buildSkillsTags(p).

לא למחוק את הפונקציות הקיימות אם הן עדיין משמשות fallback.

6. CSS

להוסיף CSS נקי וממוקד עבור:

.skills-group-grid
.skill-group-card
.skill-group-title
.skill-group-items

דרישות עיצוב:
- display grid
- 4 עמודות בדסקטופ/A4 אם יש מקום
- gap קטן ואחיד
- רקע לבן/בהיר
- border עדין
- border-radius עדין
- padding קומפקטי
- title בצבע הראשי של התוכנית var(--c0)
- items בגודל קריא
- התאמה למצב print
- לא לגרום לגלישה מחוץ ל־A4

9. Service Worker / Cache

חובה לבדוק ולעדכן את service-worker.js אם האתר משתמש ב־cache.
אם יש רשימת assets או גרסת cache:
- לעדכן את גרסת ה־cache
- לוודא ש־course-page.html ו־catalog-data.json נטענים מחדש
- למנוע מצב שבו הדפדפן מציג גרסה ישנה אחרי הפריסה

בסיום להחזיר סיכום קצר:
- אילו קבצים שונו
- איך נבדקה התצוגה
- האם עודכן Service Worker / Cache
