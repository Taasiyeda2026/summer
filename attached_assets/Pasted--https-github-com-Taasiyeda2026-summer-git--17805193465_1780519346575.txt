תעתיק לקודקס:

בריפו:
https://github.com/Taasiyeda2026/summer.git
המטרה:
להפוך את קובץ קטלוג התוכניות לקוד נקי ותבנית בלבד, כך שכל נתוני הקורסים יגיעו מקובץ JSON חיצוני, ולא יהיו כתובים בתוך קובץ ה־HTML/JS עצמו.
חשוב מאוד:
לא לשנות עיצוב.
לא לשנות CSS.
לא לשנות מבנה גרפי של הדפים.
לא לשנות את פריסת A4.
לא לשנות את כפתור ה־PDF.
לא לשנות את ההתנהגות הקיימת מעבר למקור הנתונים.
המטרה היא רק להפריד בין תבנית לבין נתונים.
הקובץ הקיים מבוסס על תבנית HTML של קטלוג תוכניות תעשיידע, שבה כרגע נתוני הקורסים היו מוטמעים בתוך הקוד. צריך לנקות את הקוד כך שיישארו רק פונקציות רינדור ותבנית, וכל התוכן יגיע מ־JSON.
יש לבצע:
1. ליצור קובץ נתונים חדש:
catalog-data.json
2. להעביר אליו את כל נתוני התוכניות במבנה מסודר, לדוגמה:
{
  "programs": [
    {
      "id": "program_1",
      "gefen": "",
      "title": "",
      "subtitle": "",
      "opening": "",
      "target": "",
      "sessions": "",
      "duration": "",
      "domain": "",
      "summary": "",
      "goal": "",
      "learningProcess": "",
      "skills": [],
      "outcomes": [],
      "description": "",
      "schoolValue": "",
      "syllabus": [
        {
          "meeting": "1",
          "topic": "",
          "description": ""
        }
      ]
    }
  ]
}
3. בקובץ ה־HTML/JS:
להסיר את כל מערכי הנתונים הקשיחים של הקורסים מתוך הקוד.
לא להשאיר בתוך הקוד שמות קורסים, מספרי גפ״ן, סילבוסים או תיאורי תוכניות.
הקוד צריך להכיל רק:
טעינת JSON
רינדור התוכניות
רינדור ניווט
רינדור סילבוס
רינדור כפתור PDF/הדפסה
טיפול בשגיאה אם ה־JSON לא נטען
4. הקריאה לנתונים:
המערכת צריכה לטעון את הנתונים מתוך:
catalog-data.json
באמצעות fetch.
אם הטעינה נכשלת, להציג הודעת שגיאה נקייה וברורה בממשק, למשל:
"לא ניתן לטעון את נתוני הקטלוג. בדקו שקובץ catalog-data.json קיים ותקין."
5. חשוב לשמור תאימות מלאה למבנה הקיים:
אם יש today/current rendering logic — לא לשנות.
אם יש tabs/nav לפי תוכניות — שימשיך להיווצר אוטומטית לפי programs.
אם יש סילבוס — שיילקח מתוך השדה syllabus של כל תוכנית.
אם שדה מסוים ריק — לא לשבור את העמוד; להציג ריק או להסתיר בעדינות לפי ההתנהגות הקיימת.
6. לנקות תנאים קשיחים:
להסיר כל תנאי שמבוסס על קורס מסוים, מספר גפ״ן מסוים או שם תוכנית מסוים.
לא צריך שיהיה בקוד תנאי כמו:
אם gefen === ...
אם title includes ...
או כל זיהוי תוכן קשיח אחר.
7. ליצור תבנית JSON נקייה:
אם אין כרגע נתונים אמיתיים להעביר, להשאיר בקובץ catalog-data.json תוכנית דוגמה ריקה/נקייה בלבד, כדי שהמערכת תעבוד כתבנית.
8. בדיקת Service Worker / Cache:
חובה לבדוק אם קיים service worker או מנגנון cache בריפו.
אם קיים:
לוודא ש־catalog-data.json נכלל נכון או מוחרג נכון מה-cache כך שלא ייטען קובץ ישן.
לעדכן גרסת cache אם נדרש.
לוודא שלאחר שינוי JSON הדפדפן מציג את הנתונים החדשים ולא נתונים ישנים.
אם אין service worker — לא להוסיף מנגנון חדש כבד, רק לציין בסיכום שאין SW רלוונטי.
9. בדיקות:
להריץ את הפרויקט מקומית.
לוודא שהעמוד נפתח ללא שגיאות console.
לוודא שהנתונים נטענים מתוך catalog-data.json.
לוודא שהניווט בין תוכניות עובד.
לוודא שהדפסה/ייצוא PDF עדיין עובד.
לוודא שאין טקסטים ישנים של קורסים שמוטמעים בתוך קובץ ה־HTML/JS.
לוודא שהעיצוב נראה זהה ככל האפשר למצב הקיים.
10. בסיכום העבודה:
לפרט אילו קבצים שונו.
לציין איפה נמצא catalog-data.json.
לציין האם נמצא service worker ומה נעשה לגביו.
לציין איך מוסיפים תוכנית חדשה בעתיד דרך JSON בלבד.
לא לבצע רידיזיין.
לא לשנות צבעים.
לא לשנות טיפוגרפיה.
לא לשנות Layout.
לא להוסיף מערכת ניהול.
לא להוסיף ספריות חדשות אם לא חובה.
רק הפרדת נתונים מהקוד והפיכת הקטלוג לתבנית נקייה שמוזנת מ־JSON.
